# Быстрый старт для реалистичной демонстрации

## 🎯 Цель

Создать на рабочем столе 4 иконки корпоративных приложений, которые выглядят как настоящие. При попытке открыть любое из них - система попросит войти через IAM.

## ⚡ Быстрая установка

### Вариант 1: Автоматическая установка (рекомендуется)

```bash
cd demo-apps
PREPARE-FOR-DEMO.bat
```

Этот скрипт:
1. Установит все зависимости
2. Соберёт все приложения
3. Создаст ярлыки на рабочем столе
4. (Опционально) Создаст простые иконки

### Вариант 2: Ручная установка

```bash
# 1. Установить зависимости
cd demo-apps
install-all.bat

# 2. Собрать приложения
build-all.bat

# 3. Создать ярлыки на рабочем столе
powershell -ExecutionPolicy Bypass -File create-all-shortcuts.ps1

# 4. (Опционально) Создать иконки
python create-simple-icons.py
```

## 📱 Что появится на рабочем столе

После установки на рабочем столе будут 4 ярлыка:

1. **📊 CRM Система** - фиолетовая иконка
2. **📧 Корпоративная почта** - синяя иконка
3. **💰 1С Бухгалтерия** - тёмно-синяя иконка
4. **📦 Склад** - оранжевая иконка

## 🎬 Как демонстрировать

### Шаг 1: Убедитесь, что Docker запущен

```bash
docker-compose up -d
docker ps  # Должно быть 6 контейнеров
```

### Шаг 2: Начните демонстрацию

1. **Покажите рабочий стол** с 4 иконками приложений
2. **Скажите**: "В компании есть разные системы - CRM, почта, 1С, склад"
3. **Двойной клик** по любой иконке (например, CRM)
4. **Откроется** окно входа в IAM System
5. **Скажите**: "Система требует авторизацию через единый портал"
6. **Войдите**: `admin@company.ru` / `Test123456!@`
7. **CRM откроется** автоматически!
8. **Скажите**: "Теперь я авторизован во всей системе"
9. **Откройте** другие приложения - они откроются БЕЗ повторного входа!

### Шаг 3: Покажите контроль доступа

1. **Закройте** все приложения
2. **Откройте** CRM снова
3. **Войдите** как `olga@company.ru` / `Test123456!@`
4. **Появится** "Доступ запрещён"
5. **Скажите**: "Система проверила права - у этого пользователя нет доступа к CRM"
6. **Откройте** Почту - она откроется!
7. **Скажите**: "Но к почте доступ есть - у каждой роли свои права"

## 🎨 Улучшение иконок (опционально)

Для более профессионального вида скачайте иконки:

### Где скачать:
- https://icons8.com (бесплатные иконки)
- https://www.flaticon.com (большой выбор)
- https://iconscout.com (профессиональные)

### Что искать:
- **CRM**: "crm", "customers", "chart", "analytics"
- **Почта**: "email", "mail", "envelope", "inbox"
- **1С**: "accounting", "calculator", "invoice", "finance"
- **Склад**: "warehouse", "inventory", "box", "storage"

### Как заменить:
1. Скачайте иконку (512x512 или больше, PNG)
2. Сохраните как `icon.png` в папке приложения:
   - `crm-app/icon.png`
   - `mail-app/icon.png`
   - `1c-app/icon.png`
   - `warehouse-app/icon.png`
3. Пересоберите приложения:
   ```bash
   cd crm-app && npm run build:win
   cd ../mail-app && npm run build:win
   cd ../1c-app && npm run build:win
   cd ../warehouse-app && npm run build:win
   ```
4. Пересоздайте ярлыки:
   ```bash
   powershell -ExecutionPolicy Bypass -File create-all-shortcuts.ps1
   ```

## ✅ Проверка готовности

Перед демонстрацией проверьте:

- [ ] Docker контейнеры запущены (`docker ps`)
- [ ] На рабочем столе 4 ярлыка
- [ ] Каждое приложение запускается
- [ ] IAM System открывается при клике на приложение
- [ ] После входа приложения открываются
- [ ] Контроль доступа работает (olga не может в CRM)

## 🚨 Troubleshooting

**Приложение не запускается:**
- Проверьте, что оно собрано: `demo-apps/crm-app/dist/win-unpacked/CRM Система.exe`
- Пересоберите: `cd crm-app && npm run build:win`

**IAM System не открывается:**
- Проверьте Docker: `docker ps`
- Проверьте frontend: http://localhost:3000
- Перезапустите: `docker-compose restart`

**Ярлыки не работают:**
- Пересоздайте: `powershell -ExecutionPolicy Bypass -File create-all-shortcuts.ps1`
- Проверьте пути в ярлыках

**"Доступ запрещён" для admin:**
- Убедитесь, что вы вошли под правильным пользователем
- Проверьте backend: http://localhost:8000/docs
- Проверьте seed данные
