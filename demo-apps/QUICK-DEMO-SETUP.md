# Быстрая настройка демонстрации

## 🚀 За 5 минут до демонстрации

### 1. Соберите приложения (если ещё не собраны)

```bash
cd demo-apps
rebuild-all.bat
```

Подождите ~5-10 минут, пока соберутся все 4 приложения.

### 2. Создайте ярлыки на рабочем столе

```bash
powershell -ExecutionPolicy Bypass -File create-shortcuts-fixed.ps1
```

Проверьте, что на рабочем столе появились 5 иконок:
- CRM Система
- Корпоративная почта
- 1С Бухгалтерия
- Складская система
- IAM System

### 3. Запустите Docker

```bash
cd ..
docker-compose up -d
```

Проверьте:
```bash
docker ps
```

Должно быть 6 контейнеров: db, redis, backend, frontend, celery-worker, celery-beat

### 4. Проверьте доступность

Откройте браузер:
- Frontend: http://localhost:3000
- Backend: http://localhost:8000/docs

## 🎬 Краткий сценарий

1. **Покажите проблему**: Кликните на 1С → появится форма входа → сообщение про IAM System
2. **Войдите в IAM**: Откройте IAM System → войдите как admin
3. **Запустите приложения**: Кликайте на карточки приложений в дашборде → они открываются без повторного входа
4. **Покажите SSO**: Кликните на иконку на рабочем столе → приложение откроется сразу (уже авторизованы)
5. **Покажите RBAC**: Выйдите, войдите как olga → попробуйте открыть CRM → доступ запрещён
6. **Покажите управление**: Войдите как admin → покажите разделы Пользователи, Роли, Аудит

## ⚠️ Возможные проблемы

### Docker не запускается
```bash
docker-compose down
docker-compose up -d
```

### Приложения не собрались
Проверьте, что установлен Node.js:
```bash
node --version
npm --version
```

Пересоберите конкретное приложение:
```bash
cd demo-apps/crm-app
npm install
npm run build
```

### Ярлыки не создались
Запустите PowerShell от администратора:
```bash
powershell -ExecutionPolicy Bypass -File create-shortcuts-fixed.ps1
```

### Приложение не открывается
1. Проверьте, что Docker запущен
2. Проверьте, что backend доступен: http://localhost:8000/docs
3. Перезапустите приложение
