# Новый сценарий демонстрации IAM System

## 🎯 Концепция

Показать, что приложения - это реальные корпоративные системы с собственными формами входа, но они работают только через централизованную систему IAM. Это демонстрирует:
- Реалистичность приложений (не просто заглушки)
- Централизованный контроль доступа
- Single Sign-On (один вход → доступ ко всем)
- Role-Based Access Control (разные роли → разный доступ)

## 📋 Подготовка

### 1. Соберите все приложения

```bash
cd demo-apps
rebuild-all.bat
```

### 2. Создайте ярлыки на рабочем столе

```bash
powershell -ExecutionPolicy Bypass -File create-shortcuts-fixed.ps1
```

На рабочем столе появятся 5 иконок:
- 📊 CRM Система
- 📧 Корпоративная почта
- 💰 1С Бухгалтерия
- 📦 Складская система
- 🔐 IAM System (главное приложение)

### 3. Запустите Docker

```bash
docker-compose up -d
docker ps  # Проверьте, что все 6 контейнеров запущены
```

## 🎬 Сценарий демонстрации

### Часть 1: Проблема (2 минуты)

**Рассказ**: 
"В компании есть много корпоративных систем - CRM, почта, 1С, склад. Каждая имеет свою форму входа, свои учётные данные. Сотрудники забывают пароли, IT-отдел тратит время на восстановление. Нужна централизованная система управления доступом."

**Показываете рабочий стол**:
- Вот наши корпоративные приложения (показываете 4 иконки)
- Давайте попробуем зайти в 1С

### Часть 2: Попытка прямого входа (2 минуты)

**Действия**:
1. Двойной клик по иконке "1С Бухгалтерия"
2. Открывается окно входа в стиле 1С
3. **Показываете**: "Вот форма входа, как в настоящей 1С"
4. Вводите любые данные (например: "Иванов И.И." / "password")
5. Нажимаете "OK"
6. **Появляется сообщение**: "Для доступа необходимо авторизоваться через IAM System"

**Комментарий**: 
"Приложение настоящее, но оно не пускает без авторизации через IAM System. Это и есть централизованный контроль доступа."

7. Закрываете окно 1С

### Часть 3: Вход через IAM System (3 минуты)

**Действия**:
1. Двойной клик по иконке "IAM System"
2. Открывается главное приложение
3. **Показываете**: "Это наша централизованная система управления доступом"

**Вход**:
4. Вводите: `admin@company.ru` / `Test123456!@`
5. Нажимаете "Войти"
6. **Показываете дашборд**: "Вот панель управления"

**Показываете секцию "Корпоративные приложения"**:
7. **Комментарий**: "Здесь все приложения компании, к которым у меня есть доступ"
8. Показываете 4 карточки: CRM, Почта, 1С, Склад

### Часть 4: Запуск приложений через IAM (3 минуты)

**Действия**:
1. Кликаете на карточку "CRM Система"
2. **Комментарий**: "Система проверяет мои права доступа..."
3. CRM открывается СРАЗУ, без формы входа!
4. **Комментарий**: "Смотрите - я уже авторизован! Это Single Sign-On"

**Показываете интерфейс CRM**:
5. Статистика клиентов, таблица с данными
6. Внизу показаны права доступа текущего пользователя

**Возвращаетесь в IAM System**:
7. Кликаете на "1С Бухгалтерия"
8. 1С открывается сразу, без формы входа!
9. **Комментарий**: "Один вход в IAM → доступ ко всем приложениям"

10. Аналогично открываете Почту и Склад
11. **Комментарий**: "Все приложения работают, пароль вводили только один раз"

### Часть 5: Проверка с рабочего стола (2 минуты)

**Действия**:
1. Не закрывая IAM System, кликаете по иконке "Корпоративная почта" на рабочем столе
2. Почта открывается СРАЗУ, без формы входа!
3. **Комментарий**: "Теперь приложения на рабочем столе тоже работают - я уже авторизован"

4. Закрываете все приложения
5. Кликаете по иконке "CRM Система" на рабочем столе
6. Снова появляется форма входа с сообщением про IAM System
7. **Комментарий**: "Сессия завершена - нужно снова войти через IAM"

### Часть 6: Контроль доступа (RBAC) (3 минуты)

**Действия**:
1. Открываете IAM System
2. Выходите из аккаунта admin
3. **Комментарий**: "Теперь покажу, как работает контроль доступа по ролям"

**Вход как обычный пользователь**:
4. Вводите: `olga@company.ru` / `Test123456!@`
5. **Комментарий**: "Это обычный сотрудник, не менеджер"
6. Входите в систему

**Показываете дашборд**:
7. **Комментарий**: "Смотрите - у неё доступ только к некоторым приложениям"
8. Кликаете на "CRM Система"
9. Появляется окно "Доступ запрещён"
10. **Показываете**: информацию о пользователе, его роль, сообщение об отсутствии прав

**Комментарий**: 
"Система проверила права и отказала в доступе. Но к почте доступ есть:"

11. Кликаете на "Корпоративная почта"
12. Почта открывается!
13. **Комментарий**: "У каждой роли свои права доступа"

### Часть 7: Управление в IAM System (3 минуты)

**Действия**:
1. Выходите, входите обратно как admin
2. Показываете разделы:
   - **Пользователи**: список с ролями, статусами
   - **Роли**: список ролей и их прав
   - **Приложения**: список всех корпоративных систем
   - **Аудит**: журнал всех действий

**Показываете возможности**:
3. **Блокировка пользователя**: одна кнопка → доступ закрыт везде
4. **Изменение роли**: меняем роль → меняются права доступа
5. **Аудит**: кто, когда, к чему обращался
6. **Аномалии**: система обнаруживает подозрительную активность

### Часть 8: Дополнительные возможности (2 минуты)

**Показываете**:
1. **MFA** - двухфакторная аутентификация
2. **Сессии** - управление активными сессиями
3. **Статистика** - графики входов, распределение событий

## 💡 Ответы на возможные вопросы

**Q: Как приложения узнают, что пользователь авторизован?**
A: Через JWT токен. IAM System выдаёт токен при входе, приложения проверяют его через API.

**Q: Что если пользователь забыл пароль?**
A: Есть функция восстановления через email. Администратор может сбросить пароль.

**Q: Как работает проверка прав?**
A: При запуске приложение спрашивает у IAM System: "Может ли этот пользователь получить доступ?" Система проверяет роль и права.

**Q: Можно ли интегрировать с реальными приложениями?**
A: Да! Через OAuth 2.0, SAML, или прямую API интеграцию. Показываете код в main.js.

**Q: Насколько это безопасно?**
A: JWT токены, bcrypt для паролей, MFA, аудит всех действий, обнаружение аномалий, защита от брутфорса.

**Q: Как добавить новое приложение?**
A: Создать запись в базе, настроить права для ролей, интегрировать через API. Показываете раздел "Приложения".
