# Сценарий демонстрации IAM System с SSO

## Подготовка к демонстрации

### 1. Убедитесь, что Docker контейнеры запущены

```bash
cd ..
docker-compose up -d
```

Проверьте статус:
```bash
docker ps
```

Должны работать: db, redis, backend, frontend, celery-worker, celery-beat

### 2. Установите зависимости для всех приложений

```bash
cd demo-apps
install-all.bat
```

### 3. Соберите все приложения

```bash
build-all.bat
```

### 4. Подготовьте иконки (опционально)

Для каждого приложения добавьте файл `icon.png` (512x512):
- `crm-app/icon.png` - иконка с графиком/людьми
- `mail-app/icon.png` - конверт
- `1c-app/icon.png` - документ/калькулятор  
- `warehouse-app/icon.png` - коробка/склад

После добавления иконок пересоберите приложения.

## Сценарий 1: Администратор - Полный доступ ко всем системам

**Цель**: Показать, что администратор имеет доступ ко всем приложениям

### Шаги:

1. **Запустите IAM System** (главное приложение)
   - Откройте `IAM System.exe` из папки `electron/dist/win-unpacked/`
   - Войдите как: `admin@company.ru` / `Test123456!@`
   - Покажите дашборд администратора

2. **Запустите CRM Систему**
   - Откройте `crm-app/dist/win-unpacked/CRM Система.exe`
   - Приложение автоматически откроется (уже авторизованы)
   - Покажите интерфейс CRM с клиентами и статистикой
   - Обратите внимание на права доступа внизу: "Чтение, Запись, Экспорт"

3. **Запустите Корпоративную почту**
   - Откройте `mail-app/dist/win-unpacked/Корпоративная почта.exe`
   - Приложение откроется автоматически
   - Покажите почтовый интерфейс с письмами

4. **Запустите 1С Бухгалтерию**
   - Откройте `1c-app/dist/win-unpacked/1С Бухгалтерия.exe`
   - Приложение откроется автоматически
   - Покажите интерфейс 1С с банковскими выписками
   - Обратите внимание на стиль, похожий на настоящую 1С

5. **Запустите Склад**
   - Откройте `warehouse-app/dist/win-unpacked/Склад.exe`
   - Приложение откроется автоматически
   - Покажите складской учёт с товарами

**Вывод**: Администратор имеет доступ ко всем 4 приложениям без дополнительных входов (SSO работает!)

## Сценарий 2: Обычный пользователь - Ограниченный доступ

**Цель**: Показать, что система контролирует доступ на основе ролей

### Шаги:

1. **Закройте все приложения**

2. **Запустите CRM Систему**
   - Откройте `crm-app/dist/win-unpacked/CRM Система.exe`
   - Откроется окно входа в IAM System
   - Войдите как: `olga@company.ru` / `Test123456!@` (обычный пользователь)
   - **Результат**: Появится сообщение "Доступ запрещён"
   - Покажите информацию о пользователе и его роли
   - Закройте окно

3. **Запустите Корпоративную почту**
   - Откройте `mail-app/dist/win-unpacked/Корпоративная почта.exe`
   - **Результат**: Приложение откроется успешно!
   - Покажите, что обычный пользователь имеет доступ к почте

4. **Попробуйте открыть 1С**
   - Откройте `1c-app/dist/win-unpacked/1С Бухгалтерия.exe`
   - **Результат**: "Доступ запрещён"

5. **Попробуйте открыть Склад**
   - Откройте `warehouse-app/dist/win-unpacked/Склад.exe`
   - **Результат**: "Доступ запрещён"

**Вывод**: Система контролирует доступ - обычный пользователь может работать только с почтой

## Сценарий 3: Менеджер - Средний уровень доступа

**Цель**: Показать гибкость системы ролей

### Шаги:

1. **Закройте все приложения**

2. **Войдите в IAM System**
   - Откройте IAM System
   - Войдите как: `marina@company.ru` / `Test123456!@` (менеджер)

3. **Проверьте доступ к приложениям:**
   - ✅ **CRM** - откроется (менеджеры работают с клиентами)
   - ✅ **Почта** - откроется (все имеют доступ к почте)
   - ❌ **1С** - доступ запрещён (финансы только для бухгалтеров)
   - ❌ **Склад** - доступ запрещён (только для администратора)

**Вывод**: Каждая роль имеет свой набор прав доступа

## Сценарий 4: Бухгалтер - Специализированный доступ

**Цель**: Показать, что разные роли имеют доступ к разным системам

### Шаги:

1. **Закройте все приложения**

2. **Войдите как бухгалтер**
   - Войдите как: `petr@company.ru` / `Test123456!@`

3. **Проверьте доступ:**
   - ⚠️ **CRM** - откроется, но только для чтения (без права записи)
   - ✅ **Почта** - полный доступ
   - ✅ **1С** - полный доступ (это основная система бухгалтера)
   - ❌ **Склад** - доступ запрещён

**Вывод**: Бухгалтер имеет полный доступ к финансовым системам

## Дополнительные возможности для демонстрации

### Показать в IAM System:

1. **Управление пользователями**
   - Создание нового пользователя
   - Назначение роли
   - Блокировка пользователя

2. **Управление приложениями**
   - Список всех интегрированных приложений
   - Настройка прав доступа

3. **Аудит**
   - Журнал всех действий пользователей
   - Кто, когда и к чему обращался

4. **Аномалии**
   - Система обнаружения подозрительной активности
   - Honeypot приложение "Зарплаты топ-менеджеров"

## Ключевые моменты для жюри

1. **Single Sign-On (SSO)**
   - Один вход для всех приложений
   - Не нужно вводить пароль в каждом приложении

2. **Role-Based Access Control (RBAC)**
   - Гибкая система ролей
   - Разные права для разных пользователей

3. **Централизованное управление**
   - Все настройки в одном месте
   - Легко добавлять новые приложения

4. **Безопасность**
   - Аудит всех действий
   - Обнаружение аномалий
   - MFA для критичных операций

5. **Реалистичность**
   - Приложения выглядят как настоящие корпоративные системы
   - Интерфейсы похожи на реальные (1С, CRM, почта)

## Troubleshooting

**Приложение не запускается:**
```bash
# Проверьте Docker
docker ps

# Перезапустите контейнеры
docker-compose restart
```

**Ошибка "Доступ запрещён" для admin:**
- Убедитесь, что вы вошли под правильным пользователем
- Проверьте, что backend доступен: http://localhost:8000/docs

**Приложение не открывается после входа:**
- Откройте DevTools (F12) и проверьте консоль
- Проверьте, что APP_ID в main.js соответствует ID в базе

**Нужно пересоздать базу данных:**
```bash
docker-compose down -v
docker-compose up -d
# Подождите 30 секунд для инициализации
```
