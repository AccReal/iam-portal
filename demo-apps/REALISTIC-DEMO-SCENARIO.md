# Реалистичный сценарий демонстрации

## 🎯 Главная идея

Показать, что приложения выглядят как настоящие корпоративные системы, установленные на компьютере. Пользователь пытается зайти в них, но система требует авторизацию через IAM.

## 📋 Подготовка

### 1. Соберите все приложения

```bash
cd demo-apps
install-all.bat
build-all.bat
```

### 2. Создайте иконки (опционально)

```bash
# Если установлен Python с Pillow
python create-simple-icons.py
```

Или скачайте профессиональные иконки:
- CRM: иконка с графиком/людьми
- Почта: конверт
- 1С: документ/калькулятор
- Склад: коробка

### 3. Создайте ярлыки на рабочем столе

```bash
powershell -ExecutionPolicy Bypass -File create-all-shortcuts.ps1
```

Теперь на рабочем столе будут 4 иконки:
- 📊 CRM Система
- 📧 Корпоративная почта
- 💰 1С Бухгалтерия
- 📦 Склад

### 4. Убедитесь, что Docker запущен

```bash
docker-compose up -d
docker ps  # Должно быть 6 контейнеров
```

## 🎬 Сценарий для жюри

### Часть 1: Проблема (2 минуты)

**Рассказ**: 
"В компании есть много разных систем - CRM, почта, 1С, склад. Каждая требует свой логин и пароль. Сотрудники забывают пароли, IT-отдел тратит время на восстановление. Нужно решение."

**Показываете рабочий стол**:
- Вот наши корпоративные приложения (показываете 4 иконки)
- Давайте попробуем зайти в CRM

### Часть 2: Первая попытка входа (3 минуты)

**Действия**:
1. Двойной клик по иконке "CRM Система"
2. Приложение запускается
3. Открывается окно IAM System с формой входа
4. **Комментарий**: "Система перенаправила меня на единый портал авторизации"

**Вход**:
5. Вводите: `admin@company.ru` / `Test123456!@`
6. Нажимаете "Войти"
7. **Комментарий**: "Я вхожу один раз в IAM System"
8. CRM открывается автоматически!
9. **Комментарий**: "И вот я уже в CRM - без повторного ввода пароля"

**Показываете интерфейс CRM**:
- Статистика клиентов
- Таблица с данными
- Внизу показаны мои права доступа

### Часть 3: SSO в действии (2 минуты)

**Действия**:
1. Не закрывая CRM, кликаете по иконке "Корпоративная почта"
2. Почта открывается СРАЗУ, без запроса пароля!
3. **Комментарий**: "Смотрите - я уже авторизован! Это Single Sign-On"

4. Кликаете по иконке "1С Бухгалтерия"
5. 1С тоже открывается автоматически!
6. **Комментарий**: "Один вход - доступ ко всем системам"

7. Кликаете по иконке "Склад"
8. Склад открывается
9. **Комментарий**: "Все приложения работают с единой системой авторизации"

### Часть 4: Контроль доступа (3 минуты)

**Действия**:
1. Закрываете все приложения
2. **Комментарий**: "Теперь покажу, как система контролирует доступ"

3. Кликаете по иконке "CRM Система"
4. Открывается форма входа
5. Вводите: `olga@company.ru` / `Test123456!@` (обычный пользователь)
6. **Комментарий**: "Это обычный сотрудник, не менеджер"

7. Появляется красивое окно "Доступ запрещён"
8. **Показываете**:
   - Информацию о пользователе
   - Его роль
   - Сообщение о том, что нет прав
9. **Комментарий**: "Система проверила права и отказала в доступе"

10. Закрываете окно, кликаете по "Корпоративная почта"
11. Почта открывается!
12. **Комментарий**: "Но к почте доступ есть - у каждой роли свои права"

### Часть 5: Управление в IAM System (2 минуты)

**Действия**:
1. Открываете IAM System (главное приложение)
2. Входите как admin
3. Показываете:
   - Список пользователей с ролями
   - Список приложений
   - Матрицу прав доступа
   - Журнал аудита

4. **Комментарий**: "Администратор управляет всем из одного места"

### Часть 6: Дополнительные возможности (2 минуты)

**Показываете в IAM System**:
1. **Аудит** - кто, когда, к чему обращался
2. **Аномалии** - система обнаруживает подозрительную активность
3. **MFA** - двухфакторная аутентификация для критичных операций
4. **Блокировка** - можно заблокировать пользователя одной кнопкой

## 💡 Ответы на возможные вопросы

**Q: Как добавить новое приложение?**
A: Создать запись в базе, настроить права, интегрировать через API. Показываете раздел "Приложения" в IAM System.

**Q: Что если пользователь забыл пароль?**
A: Есть функция восстановления через email. Администратор может сбросить пароль.

**Q: Как работает проверка прав?**
A: При каждом обращении приложение спрашивает у IAM System: "Может ли этот пользователь получить доступ?" Система проверяет роль и права.

**Q: Можно ли интегрировать с реальными приложениями?**
A: Да! Через OAuth 2.0, SAML, или прямую API интеграцию. Показываете код в main.js.

**Q: Насколько это безопасно?**
A: JWT токены, bcrypt для паролей, MFA, аудит всех действий, обнаружение аномалий.
