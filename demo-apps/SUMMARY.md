# Demo Applications - Summary

## Что создано

### 4 демонстрационных desktop приложения:

1. **CRM Система** (`crm-app/`)
   - Современный интерфейс управления клиентами
   - Статистика продаж и активных сделок
   - Таблица клиентов с контактами
   - Цветовая схема: фиолетовый градиент

2. **Корпоративная почта** (`mail-app/`)
   - Почтовый клиент с боковой панелью
   - Список писем с превью
   - Папки: Входящие, Важные, Отправленные, Черновики
   - Цветовая схема: синий

3. **1С Бухгалтерия** (`1c-app/`)
   - Интерфейс в стиле настоящей 1С
   - Банковские выписки и операции
   - Статистика по счетам
   - Цветовая схема: тёмно-синий (фирменный 1С)

4. **Складская система** (`warehouse-app/`)
   - Управление товарами на складе
   - Статусы: В наличии, Мало, Нет в наличии
   - Статистика по складу
   - Цветовая схема: оранжевый

## Как работает SSO

```
1. Пользователь запускает демо-приложение
         ↓
2. Приложение проверяет наличие токена авторизации
         ↓
3. Если токена нет → открывает IAM System для входа
         ↓
4. Пользователь входит в IAM System
         ↓
5. IAM System сохраняет токен
         ↓
6. Демо-приложение получает токен
         ↓
7. Проверяет права доступа через Backend API
         ↓
8. Если есть доступ → показывает интерфейс
   Если нет → показывает "Доступ запрещён"
```

## Технические детали

### Структура каждого приложения:

```
app-name/
├── package.json          # Конфигурация Electron
├── main.js              # Логика SSO и проверки прав
├── app.html             # Интерфейс приложения
├── access-denied.html   # Страница отказа в доступе
├── icon.png             # Иконка приложения
└── README.md            # Документация
```

### Ключевые функции в main.js:

- `checkAuth()` - проверка авторизации пользователя
- `checkAccess(token)` - проверка прав доступа к приложению
- `createAuthWindow()` - открытие IAM System для входа
- `createMainWindow(user)` - открытие интерфейса приложения
- `showAccessDenied(user)` - показ сообщения об отказе

## Что показывать жюри

### 1. Единый вход (SSO)
- Запустить IAM System
- Войти как admin
- Запустить любое демо-приложение
- Оно откроется автоматически без повторного входа

### 2. Контроль доступа (RBAC)
- Выйти из всех приложений
- Войти как обычный пользователь (olga@company.ru)
- Попытаться открыть CRM → "Доступ запрещён"
- Открыть Почту → успешно

### 3. Разные роли
- Показать, что менеджер имеет доступ к CRM, но не к 1С
- Показать, что бухгалтер имеет доступ к 1С, но не к складу
- Показать, что только админ имеет доступ ко всему

### 4. Реалистичность
- Интерфейсы выглядят как настоящие корпоративные системы
- 1С действительно похожа на 1С
- CRM выглядит современно и профессионально
- Почта похожа на Outlook/Gmail

## Преимущества для компании

1. **Удобство** - один вход для всех систем
2. **Безопасность** - централизованный контроль доступа
3. **Гибкость** - легко добавлять новые приложения
4. **Аудит** - все действия логируются
5. **Масштабируемость** - работает с любым количеством приложений
